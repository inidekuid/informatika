Jadi, Apa itu Algoritma?

Secara sederhana, algoritma adalah serangkaian langkah logis dan berurutan untuk menyelesaikan suatu masalah.

Anggap saja seperti resep membuat mi instan: ada urutan yang jelas untuk mendapatkan hasil yang sama setiap saat.

Hari ini, kita akan fokus pada "resep" yang digunakan komputer untuk mencari data.

Fokus Kita: Algoritma Pencarian

Algoritma Pencarian adalah "resep" khusus untuk menemukan sebuah data spesifik di dalam sekumpulan data yang besar.

Tujuannya sederhana: menemukan apa yang kita cari secepat dan seefisien mungkin.

Ini seperti mencari satu buku di perpustakaan raksasa. Kita butuh cara yang cerdas!

Konsep Kunci: Array & Indeks

Komputer menyimpan kumpulan data (misal: daftar angka) dalam sebuah "wadah" yang disebut Array.

Setiap posisi dalam array memiliki alamat unik yang disebut Indeks.

PENTING: Indeks di dunia pemrograman SELALU dimulai dari angka 0, bukan 1!

Contoh: Jika kita punya 10 data, maka indeksnya adalah dari 0 sampai 9.

Indeks 0 1 2 3 4 ... 9
Data Data ke-1 Data ke-2 Data ke-3 Data ke-4 Data ke-5 ... Data ke-10

Dua "Resep" Utama Mencari Data

Linear Search

Memeriksa data satu per satu dari awal (indeks 0, 1, 2, ...). Sederhana tapi bisa lambat.

üîé

Analogi: Mencari kunci di tas berantakan.

Binary Search

Memotong data jadi dua berulang kali. Super cepat, tapi data harus urut.

üìñ

Analogi: Mencari kata di kamus.

Contoh: Mencari Angka 25 dengan Linear Search

Kita cek setiap kotak satu per satu dari indeks 0.

010
15
230
315
425
520

Langkah 1: Cek indeks 0 (nilai 10). Bukan 25.

Langkah 2: Cek indeks 1 (nilai 5). Bukan 25.

Langkah 3: Cek indeks 2 (nilai 30). Bukan 25.

Langkah 4: Cek indeks 3 (nilai 15). Bukan 25.

Langkah 5: Cek indeks 4 (nilai 25). Ditemukan!

Contoh: Mencari Angka 25 dengan Binary Search

Data wajib urut! Kita mulai dari tengah.

05
110
215
320
425
530

Langkah 1: Cek nilai tengah (indeks 2, nilai 15). Karena 25 > 15, kita buang separuh bagian kiri dan hanya mencari di bagian kanan.

Langkah 2: Cek nilai tengah dari sisa data (indeks 4, nilai 25). Ditemukan!

Jauh lebih sedikit langkah dibandingkan Linear Search!

Perbedaan Utama

Linear Search

  • Kecepatan: Lambat pada data besar.
  • Syarat: Data tidak perlu terurut.
  • Kekuatan: Kesederhanaannya.

Binary Search

  • Kecepatan: Sangat cepat.
  • Syarat: Data WAJIB terurut.
  • Kekuatan: Efisiensinya.

Kapan Menggunakannya?

Gunakan Linear Search jika...

  • Datanya sedikit.
  • Datanya acak / tidak terurut.
  • Contoh: Mencari baju di tumpukan lemari.

Gunakan Binary Search jika...

  • Datanya sangat banyak.
  • Datanya pasti terurut.
  • Contoh: Mencari kontak di HP atau lagu di Spotify.

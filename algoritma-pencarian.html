Bab 1: Pengantar Algoritma

Apa Itu Algoritma?

Bayangkan kamu ingin membuat mi instan. Pasti ada langkah-langkah yang harus kamu ikuti, kan?

  1. Rebus air hingga mendidih.
  2. Masukkan mi ke dalam air mendidih.
  3. Sambil menunggu, siapkan bumbu di mangkuk.
  4. Setelah mi matang, tiriskan dan tuang ke mangkuk.
  5. Aduk hingga rata dan mi siap dimakan.

Langkah-langkah yang berurutan dan logis untuk menyelesaikan suatu masalah inilah yang disebut algoritma. Dalam dunia komputer, algoritma adalah "resep" yang kita berikan kepada komputer untuk melakukan suatu tugas.

Bab 2: Fokus Kita: Algoritma Pencarian

Apa Itu Algoritma Pencarian?

Algoritma Pencarian (Search Algorithm) adalah serangkaian langkah sistematis yang digunakan untuk menemukan informasi atau data tertentu dari sekumpulan data.

Mengapa Ini Penting?

Setiap hari, kita melakukan pencarian. Efisiensi dalam menemukan sesuatu sangat menghemat waktu dan tenaga.

  • Contoh Kehidupan Sehari-hari (Unplugged):
    • Mencari nama teman di daftar absen kelas.
    • Mencari kata di kamus.
    • Mencari buku di rak perpustakaan yang sudah diurutkan berdasarkan kategori.
  • Contoh di Dunia Digital (Plugged):
    • Mencari kontak di ponsel.
    • Menggunakan fitur "Find" (Ctrl+F) di dokumen.
    • Mencari produk di toko online.
    • Mesin pencari seperti Google menggunakan algoritma pencarian yang sangat kompleks untuk menemukan halaman web yang relevan.

Tanpa algoritma pencarian yang efisien, tugas-tugas di atas akan memakan waktu sangat lama!

Bab 3: Linear Search (Pencarian Lurus)

Pengertian

Linear Search adalah metode pencarian paling dasar. Cara kerjanya adalah dengan memeriksa setiap elemen dalam kumpulan data satu per satu, dari awal sampai akhir, hingga elemen yang dicari ditemukan atau sampai seluruh data selesai diperiksa.

Analogi: Kamu kehilangan kunci di dalam tas. Kamu akan mengeluarkan isi tas satu per satu sampai kunci itu ketemu.

Langkah Kerja Algoritma

Misalkan kita punya daftar angka: [12, 5, 33, 29, 8, 41] dan kita ingin mencari angka 29.

  1. Mulai dari elemen pertama (indeks 0), yaitu 12. Apakah 12 sama dengan 29? Bukan.
  2. Lanjut ke elemen kedua (indeks 1), yaitu 5. Apakah 5 sama dengan 29? Bukan.
  3. Lanjut ke elemen ketiga (indeks 2), yaitu 33. Apakah 33 sama dengan 29? Bukan.
  4. Lanjut ke elemen keempat (indeks 3), yaitu 29. Apakah 29 sama dengan 29? YA!
  5. Pencarian berhenti. Angka 29 ditemukan pada indeks ke-3.

Pseudocode

FUNCTION linearSearch(data, target)
  UNTUK setiap elemen DALAM data:
    JIKA elemen == target MAKA
      KEMBALIKAN "Data ditemukan pada posisi" + posisi saat ini
    AKHIR JIKA
  AKHIR UNTUK

  KEMBALIKAN "Data tidak ditemukan"
AKHIR FUNCTION

Kelebihan dan Kekurangan

  • Kelebihan: Sangat sederhana, mudah dipahami, dan tidak memerlukan data terurut.
  • Kekurangan: Sangat tidak efisien untuk data besar.

Contoh Kode (Python)

def linear_search(data_list, target):
  """Mencari target dalam list menggunakan Linear Search."""
  for i in range(len(data_list)):
    if data_list[i] == target:
      return f"Angka {target} ditemukan di indeks {i}."
  return f"Angka {target} tidak ditemukan dalam daftar."

# Contoh penggunaan
angka = [12, 5, 33, 29, 8, 41]
print(linear_search(angka, 29)) # Output: Angka 29 ditemukan di indeks 3.

Bab 4: Binary Search (Pencarian Biner)

Pengertian dan Syarat Utama

Binary Search adalah algoritma pencarian yang jauh lebih cepat, tetapi memiliki satu syarat mutlak: DATA HARUS SUDAH TERURUT (dari kecil ke besar atau sebaliknya).

Analogi: Mencari nama "Rudi" di kamus. Kamu tidak akan mulai dari huruf A. Kamu akan langsung membuka kira-kira di tengah (huruf M), lalu memutuskan mencari di separuh awal atau akhir kamus, dan mengulangi prosesnya.

Langkah Kerja Algoritma

Misalkan kita punya daftar angka yang sudah terurut: [4, 8, 12, 16, 23, 28, 35, 41] dan kita ingin mencari angka 12.

  1. Tentukan batas kiri (indeks 0) dan kanan (indeks 7).
  2. Cari posisi tengah: (0 + 7) / 2 -> indeks 3 (nilai 16).
  3. Bandingkan target (12) dengan nilai tengah (16). Karena 12 < 16, buang bagian kanan. Batas kanan sekarang menjadi indeks 2.
  4. Data sisa: [4, 8, 12]. Cari tengah baru: (0 + 2) / 2 -> indeks 1 (nilai 8).
  5. Bandingkan target (12) dengan nilai tengah (8). Karena 12 > 8, buang bagian kiri. Batas kiri sekarang menjadi indeks 2.
  6. Data sisa: [12]. Cari tengah baru: (2 + 2) / 2 -> indeks 2 (nilai 12).
  7. Bandingkan target (12) dengan nilai tengah (12). SAMA! Data ditemukan.

Pseudocode

FUNCTION binarySearch(data, target)
  kiri = 0
  kanan = panjang(data) - 1

  SELAMA kiri <= kanan:
    tengah = (kiri + kanan) / 2
    
    JIKA data[tengah] == target MAKA
      KEMBALIKAN "Data ditemukan"
    JIKA data[tengah] < target MAKA
      kiri = tengah + 1  // Cari di kanan
    LAIN
      kanan = tengah - 1 // Cari di kiri
    AKHIR JIKA
  AKHIR SELAMA

  KEMBALIKAN "Data tidak ditemukan"
AKHIR FUNCTION

Kelebihan dan Kekurangan

  • Kelebihan: Sangat cepat dan efisien untuk data besar.
  • Kekurangan: Wajib data terurut, implementasi sedikit lebih kompleks.

Contoh Kode (Python)

def binary_search(data_list, target):
  """Mencari target dalam list terurut menggunakan Binary Search."""
  kiri, kanan = 0, len(data_list) - 1

  while kiri <= kanan:
    tengah = (kiri + kanan) // 2
    nilai_tengah = data_list[tengah]

    if nilai_tengah == target:
      return f"Angka {target} ditemukan di indeks {tengah}."
    elif nilai_tengah < target:
      kiri = tengah + 1
    else:
      kanan = tengah - 1
      
  return f"Angka {target} tidak ditemukan."

# Contoh penggunaan
angka_urut = [4, 8, 12, 16, 23, 28, 35, 41]
print(binary_search(angka_urut, 12)) # Output: Angka 12 ditemukan di indeks 2.

Bab 5: Kesimpulan - Linear vs. Binary

Kriteria Linear Search Binary Search
Kecepatan Lambat pada data besar Sangat Cepat pada data besar
Syarat Data Tidak perlu diurutkan Wajib sudah terurut
Kompleksitas Sederhana Sedang
Kapan Digunakan? Saat data sedikit, acak, atau ketika kita tidak yakin data sudah terurut. Saat data sangat besar dan kita yakin data sudah dalam keadaan terurut.
